# Homeflowers.shop บ้านดอกไม้
ระบบร้านค้าออนไลน์สำหรับจำหน่ายดอกไม้แห้งและดอกไม้ประดิษฐ์

# หลักการและเหตุผล
- ดอกไม้แห้งและดอกไม้ประดิษฐ์ค่อนข้างเป็นที่นิยมมากในขณะนี้ นิยมให้กันในโอกาสพิเศษต่างๆ เพราะสามารถเก็บได้นาน ไม่เหี่ยว จัดส่งง่าย ดูแลง่ายด้วย
- จากความนิยม ผู้จัดทำจึงมีแนวคิดในการพัฒนาระบบพาณิชย์อิเล็กทรอนิกส์ (e-Commerce) สำหรับร้านจำหน่ายดอกไม้แห้ง/ประดิษฐ์ เพื่อเป็นช่องทางในการนำเสนอสินค้า เพิ่มความสะดวกในการเลือกซื้อสินค้า การสั่งซื้อ การชำระเงิน และการติดตามสถานะคำสั่งซื้อผ่านระบบออนไลน์ อีกทั้งยังช่วยให้ผู้ประกอบการสามารถบริหารจัดการข้อมูลสินค้า คำสั่งซื้อ และข้อมูลลูกค้าได้อย่างมีประสิทธิภาพ ส่งผลให้การดำเนินธุรกิจมีความสะดวก รวดเร็ว และสามารถขยายฐานลูกค้าได้มากยิ่งขึ้น

# วัตถุประสงค์
- เพื่อวิเคราะห์และออกแบบระบบซื้อขายดอกไม้แห้ง ดอกไม้ประดิษฐ์
- เพื่อพัฒนาระบบซื้อขายดอกไม้แห้ง ดอกไม้ประดิษฐ์
- เพื่อทดสอบและประเมินผลจากการใช้งานระบบซื้อขายดอกไม้แห้ง ดอกไม้ประดิษฐ์ที่เราพัฒนา

# ขอบเขตของระบบ
จะประกอบด้วยกระบวนการทำงาน 3 ส่วน ได้แก่
- 1.ลูกค้า (Customer)
- ลูกค้าสามารถสมัครสมาชิก เข้าสู่ระบบและจัดการแก้ไขข้อมูลส่วนตัว
- ลูกค้าสามารถค้นหาและแสดงรายการสินค้าตามความต้องการ
- ลูกค้าสามารถสร้างตะกร้าสินค้า และจัดการข้อมูลในตะกร้าสินค้า
- ลูกค้าสามารถสั่งซื้อสินค้า เลือกรูปแบบการชำระเงิน และการจัดส่ง
- ลูกค้าสามารถตรวจสอบคำสั่งซื้อ และติดตามสถานะคำสั่งซื้อได้

- 2.พนักงาน (Straff)
- จัดการข้อมูลสินค้า (CRUD)
- จัดการคำสั่งซื้อ ยืนยันการชำระเงิน และอัปเดตสถานะการจัดส่ง
- จัดการข้อมูลสมาชิก ตรวจสอบข้อมูล และลบข้อมูลสมาชิกได้
  
- 3.ผู้ดูแลระบบ (Admin)
- จัดการข้อมูลสินค้า (CRUD)
- จัดการคำสั่งซื้อ ยืนยันการชำระเงิน และอัปเดตสถานะการจัดส่ง
- จัดการข้อมูลสมาชิก ตรวจสอบข้อมูล และลบข้อมูลสมาชิกได้
- จัดการข้อมูลพนักงาน รวมถึงกำหนดสิทธิ์การใช้งาน (Staff Role) เพื่อให้สามารถเข้าถึงฟังก์ชันที่เกี่ยวข้องตามหน้าที่ที่รับผิดชอบ


# แนวทางการพัฒนาตาม SDLC
- ในโครงงานนี้มีขั้นตอน 5 ขั้นตอน ดังนี้
- Planing : ศึกษาความเป็นไปได้ของโครงงาน กำหนดวัตถุประสงค์ ขอบเขต และความต้องการของระบบ รวมถึงวางแผนระยะเวลาการดำเนินงาน ทรัพยากร และแบ่งหน้าที่ของสมาชิกในทีม เพื่อให้การพัฒนาระบบเป็นไปตามแผนที่กำหนด
- Analysis : วิเคราะห์ความต้องการของผู้ใช้งานทั้งลูกค้าและผู้ดูแลระบบ ศึกษาปัญหาและกระบวนการทำงานของร้านขายดอกไม้แห้ง/ประดิษฐ์ เพื่อนำมากำหนดความสามารถของระบบ เช่น การสมัครสมาชิก การจัดการสินค้า การสั่งซื้อ การชำระเงิน และการติดตามสถานะคำสั่งซื้อ
- Design : ออกแบบ UX/UI , ออกแบบการทำงานของระบบ เช่น Use Case Diagram, Class Diagram , Sequence Diagram และโครงสร้างของระบบ เพื่อให้ระบบมีความเป็นระเบียบและรองรับการใช้งานได้อย่างมีประสิทธิภาพ
- Development : พัฒนาระบบตามแบบที่ออกแบบไว้ โดยแบ่งเป็นส่วน Frontend สำหรับการแสดงผลเว็บไซต์ และ Backend สำหรับจัดการข้อมูลและประมวลผล พร้อมเชื่อมต่อฐานข้อมูลและพัฒนาฟังก์ชันต่าง ๆ เช่น การจัดการสินค้า ตะกร้าสินค้า การสั่งซื้อ การชำระเงิน และระบบผู้ดูแล
- Testing : ทดสอบการทำงานของระบบในทุกฟังก์ชัน ตรวจสอบความถูกต้องของข้อมูล ความปลอดภัย และประสิทธิภาพการทำงาน รวมถึงทดสอบการใช้งานจริง เพื่อค้นหาและแก้ไขข้อผิดพลาดก่อนนำระบบไปใช้งาน


# เครื่องมือและเทคโนโลยีที่ใช้
- Frontend : HTML / CSS / Javascript / React / Bootstrap
- Backend : Node.js
- Database : Localstorage
- Design tool : Figma
- Version Control : Github

# แนวทางการทดสอบระบบ
- ประเภทการทดสอบ : User acceptance Testing (UAT) เนื่องจากทำงานร่วมกัน
- เครื่องมือที่ใช้ : Manual Testing
- รายละเอียดการทดสอบ : *** ไม่วัดผลการใช้เครื่องมือทดสอบอัตโนมัติหรือมีการจัดทำรายงานผลการทดสอบอย่างเป็นทางการ *** การทดสอบการทำงานของระบบด้วยตนเอง ตามฟังก์ชันที่พัฒนาพร้อมสาธิตการทำงานต่อผู้สอน โดยอธิบายขั้นตอนการทดสอบผลลัพธ์ที่คาดหวังและผลลัพธ์ที่เกิดขึ้นจริง เพื่อแสดงให้เห็นว่าระบบทำงานได้ถูกต้องตามวัตถุประสงค์ที่กำหนดไว้

  
# ผลลัพท์ที่คาดว่าจะได้รับ
- ช่วยเพิ่มช่องทางการจำหน่ายดอกไม้แห้ง/ดอกไม้ประดิษฐ์ ผ่านระบบออนไลน์ ทำให้เข้าถึงลูกค้าได้กว้างขวางมากยิ่งขึ้น
- ช่วยอำนวยความสะดวกให้ลูกค้าสามารถเลือกชมสินค้า สั่งซื้อ และชำระเงินได้ทุกที่ทุกเวลา ผ่านเว็บไซต์
- ช่วยจัดเก็บข้อมูลสินค้า ข้อมูลลูกค้า และข้อมูลการสั่งซื้ออย่างเป็นระบบ ทำให้สามารถค้นหาและนำข้อมูลมาใช้งานได้อย่างรวดเร็ว
- ช่วยให้ผู้ดูแลระบบสามารถติดตามและจัดการคำสั่งซื้อ รวมถึงตรวจสอบสถานะการชำระเงินได้อย่างมีประสิทธิภาพ
- สามารถนำระบบไปต่อยอดเพิ่มเติมในอนาคตได้ เช่น ระบบรีวิวสินค้า ระบบคูปองส่วนลด หรือระบบการเชื่อมต่อช่องทางการชำระเงินออนไลน์

  
# แผนการดำเนินงาน 4 สัปดาห์

| สัปดาห์          | กิจกรรม                | รายละเอียด                                                                                                                               |
| ---------------- | ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| **สัปดาห์ที่ 1** | วิเคราะห์ระบบและออกแบบ | ศึกษาความต้องการของผู้ใช้งาน วิเคราะห์ระบบ ออกแบบหน้าจอผู้ใช้งาน (UI/UX) และจัดทำแผนผังการทำงานของระบบ |
| **สัปดาห์ที่ 2** | พัฒนา Frontend         | พัฒนาหน้าเว็บไซต์สำหรับผู้ใช้งาน เช่น หน้าแรก หน้าสินค้า หน้ารายละเอียดสินค้า หน้าตะกร้าสินค้า และหน้าเข้าสู่ระบบ |
| **สัปดาห์ที่ 3** | พัฒนา Backend          | พัฒนาระบบฐานข้อมูล รวมถึงฟังก์ชันจัดการสมาชิก สินค้า คำสั่งซื้อ การชำระเงิน และระบบผู้ดูแล (Admin) |
| **สัปดาห์ที่ 4** | ทดสอบระบบและนำเสนอ     | ทดสอบการทำงานของระบบ แก้ไขข้อผิดพลาด จัดทำรายงานนำเสนอ พร้อมสาธิตการใช้งานระบบ |
